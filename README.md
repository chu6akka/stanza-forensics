# Stanza Forensics GUI

Простая программа с графическим интерфейсом (Tkinter) для судебно-лингвистического анализа русского текста.

## Важное изменение

По умолчанию backend теперь **Natasha** (стабильнее в Windows-средах), а **Stanza** оставлена как дополнительный режим.

Почему так:
- у части пользователей Stanza не запускается из-за несовместимости окружения (чаще всего Python/Torch/модель);
- Natasha в таких условиях обычно запускается без проблем и дает качественный морфологический разбор.

## Что теперь в приложении

- переключатель backend: `natasha` / `stanza`;
- если выбрана `stanza`, но она не стартует — приложение автоматически откатывается на `natasha`;
- в интерфейсе явно показывается, какой backend реально используется (`backend: ...`);
- для слова в таблице показываются русские морфологические признаки;
- при наведении на строку таблицы слово подсвечивается в исходном тексте;
- есть кнопка круговой диаграммы долей частей речи.

## Про проблему «stanza не работает»

Если вы выбираете stanza, но видите в итоге `backend: natasha`, это значит, что в текущей среде stanza не инициализировалась (пакет/модель/torch-окружение). Это не ошибка вашего текста — это ограничение окружения.

Рекомендации для stanza:
- Python 3.10–3.11 (обычно наиболее беспроблемно);
- отдельное virtualenv;
- установить зависимости заново и дать stanza скачать RU-модель при первом запуске.

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Запуск

```bash
python pos_forensics.py
```
